# Сборка и публикация на Яндекс Играх

Пошаговая инструкция: от чистого проекта в Unity до опубликованной игры на
games.yandex.ru.

## 1. Подготовка в Unity Editor (один раз)

1. Открыть проект в Unity.
2. Переключить платформу сборки:
   `File > Build Settings > WebGL > Switch Platform`.
3. Выбрать шаблон Yandex Games:
   `Project Settings > Player > Resolution and Presentation > WebGL Template`
   → **YandexGames**.
4. В `Player Settings > Publishing Settings`:
   - **Compression Format** → `Disabled` (самый простой и надёжный вариант).
     Если важен размер архива — можно `Gzip`/`Brotli`, но тогда обязательно
     включить **Decompression Fallback**.
   - Остальные настройки (Data caching и т.д.) можно оставить по умолчанию.
5. Проверить версию игры: `Player Settings > Other Settings > Version` —
   значение попадёт в `v{version}` на тайтл-экране.

## 2. Сборка

1. `File > Build Settings > Build`.
2. Указать пустую папку, например `Build/WebGL`.
3. Дождаться окончания сборки. Внутри должно получиться:
   ```
   Build/WebGL/
     index.html
     Build/
       *.data(.br|.gz)
       *.framework.js(.br|.gz)
       *.wasm(.br|.gz)
       *.loader.js
     StreamingAssets/ (если есть)
   ```

## 3. Локальная проверка перед загрузкой

Игра должна нормально запускаться и вне Яндекс Игр (SDK просто не найдётся,
`YandexGamesManager` залогирует предупреждение и продолжит работу в
"мок"-режиме).

```bash
cd Build/WebGL
python -m http.server 8000
```

Открыть `http://localhost:8000` и убедиться, что:
- игра загружается и тайтл-экран отображается;
- нет ошибок в консоли браузера (F12 → Console);
- реклама на старте молча пропускается (SDK недоступен локально — это ок).

## 4. Упаковка в архив

Архивируется **содержимое** папки сборки, а не сама папка — `index.html`
должен лежать в корне zip-файла.

```bash
cd Build/WebGL
zip -r ../../release.zip .
```

Проверить: `unzip -l release.zip` — первой строкой должен быть `index.html`,
а не `WebGL/index.html`.

## 5. Загрузка в Консоль разработчика

1. Открыть [Консоль Яндекс Игр](https://games.yandex.ru/console).
2. Если игры ещё нет — создать черновик (**Добавить игру**).
3. На вкладке загрузки билда выбрать `release.zip` и дождаться обработки.
4. Проверить билд через **debug-панель**:
   - В консоли нажать **Открыть с debug-панелью**, либо добавить
     `?debug-mode=16` к ссылке игры.
   - В левом нижнем углу индикатор лоадера должен показать `IT`
     (SDK-загрузчик подключен верно). Если `IF` — значит, скрипт SDK
     подключается по устаревшему пути (см. `Assets/WebGLTemplates/YandexGames/index.html`,
     должен быть `<script src="/sdk.js">`).
   - Убедиться, что реклама при запуске и при проигрыше реально показывается
     (в debug-режиме на пустом инвентаре рекламы Яндекс обычно показывает
     тестовый ролик-заглушку).

## 6. Заполнение карточки игры (черновик в Консоли)

Прежде чем отправлять на модерацию, заполнить:
- **Название, короткое и полное описание** (на русском и, по желанию, других языках).
- **Иконка** и **обложка** — точные требования по размеру смотреть в Консоли
  при загрузке (там прямо указываются актуальные пиксельные размеры).
- **Скриншоты** (минимум несколько, показывающих реальный геймплей).
- **Категория** и **возрастной тег** — исходя из содержания игры (насилие в
  игре мультяшное/условное — обычно подходит "0+"/"6+", но финальное решение
  принимает Консоль/модерация).
- При желании — **монетизация**: вкладка **Реклама** в Консоли, подключить
  Рекламную сеть Яндекса (РСЯ), если нужен доход с рекламы сверх того, что
  уже показывается через `ysdk.adv.*` в коде.

## 7. Отправка на модерацию

1. В черновике нажать **Отправить на модерацию**.
2. Модерация обычно занимает от нескольких часов до пары дней.
3. Если отклонят — в Консоли будет указана причина; типовые причины см. в
   `YANDEX_GAMES.md` (раздел "Still needs attention before submitting").

## 8. Выпуск обновлений

Для каждого следующего релиза:
1. Поднять версию в `Player Settings > Version` (по желанию — на усмотрение).
2. Пересобрать WebGL (шаги 2–4 выше).
3. В Консоли на странице игры загрузить новый zip как новую версию билда.
4. Проверить через debug-панель.
5. Опубликовать — в зависимости от настроек Консоли это либо публикуется
   сразу, либо тоже уходит на повторную модерацию (обычно более быстро, чем
   первая публикация).

## Частые проблемы

| Симптом | Причина | Решение |
|---|---|---|
| `Uncaught ReferenceError: YaGames is not defined` | Скрипт SDK не успел загрузиться до вызова `YaGames.init()` | Проверить, что в `index.html` тег `<script src="/sdk.js">` стоит до основного скрипта загрузки Unity |
| Индикатор лоадера `IF` вместо `IT` | Используется устаревший/неверный путь к SDK | Путь должен быть относительным `/sdk.js` при загрузке через Консоль |
| Реклама не показывается локально | SDK недоступен вне Яндекс Игр | Это ожидаемо — тестировать рекламу нужно через debug-панель в Консоли |
| Игра "мигает"/не на весь экран | Не выбран WebGL-шаблон **YandexGames** | `Project Settings > Player > WebGL Template` → YandexGames, пересобрать |
| Модерация отклонила из-за скролла/выделения текста | Правки применились не полностью | Убедиться, что билет собран уже после наших правок `index.html`, а не из старого билда |
